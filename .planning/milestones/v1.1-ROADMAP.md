## Milestone Summary

**Key Decisions:**
- Package name: q-kangaroo (PyPI) / q_kangaroo (import) -- user-chosen
- Rust crate names unchanged (qsym-core, qsym-python) -- only Python-facing names renamed
- ABI3 stable ABI via maturin features (not Cargo.toml) for cross-version wheels
- OIDC trusted publishing for PyPI (no stored API tokens)
- GitHub Pages for docs via peaceiris/actions-gh-pages
- Furo theme + autodoc + napoleon for Sphinx documentation
- Explicit session parameter kept as primary API (get_default_session for convenience)
- Pre-executed notebooks (nbsphinx_execute="never") for reliable docs builds

**Issues Resolved:**
- qsymbolic -> q_kangaroo rename completed cleanly (zero test regressions)
- DLL bundling for Windows wheels (libgmp-10, libgcc_s_seh-1, libwinpthread-1)
- ABI3 feature conflicts resolved via maturin features instead of Cargo.toml

**Issues Deferred:**
- macOS CI support (deferred to v2)
- Static GMP linking on Windows (eliminate DLL dependency)
- OWNER placeholder replacement (user does before first publish)

**Technical Debt Incurred:**
- Placeholder author/owner fields in pyproject.toml and README badge URLs
- Notebooks pre-executed locally, not in CI (fragile if API changes)
