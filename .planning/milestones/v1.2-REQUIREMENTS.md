## v1.2 Requirements

### Polynomial Infrastructure

- [x] **POLY-01**: Dense univariate polynomial type (QRatPoly) with add, sub, mul, div, rem over QRat coefficients -- Phase 13
- [x] **POLY-02**: Polynomial GCD via subresultant PRS with content extraction to prevent coefficient explosion -- Phase 13
- [x] **POLY-03**: Polynomial resultant computation -- Phase 13
- [x] **POLY-04**: q-shift operations: evaluate p(q*x), p(q^j*x) for polynomial p -- Phase 13
- [x] **POLY-05**: Rational function type (QRatRationalFunc) with arithmetic and simplification -- Phase 13

### q-Gosper Algorithm

- [x] **GOSP-01**: q-hypergeometric term ratio extraction from HypergeometricSeries as rational function of q^k -- Phase 14
- [x] **GOSP-02**: q-dispersion computation via resultant (find all j where gcd(a(x), b(q^j*x)) != 1) -- Phase 14
- [x] **GOSP-03**: q-Greatest Factorial Factorization (qGFF) decomposition into sigma, tau, p -- Phase 14
- [x] **GOSP-04**: Key equation solver (find polynomial f satisfying sigma(x)*f(qx) - tau(x)*f(x) = p(x)) -- Phase 14
- [x] **GOSP-05**: Complete q-Gosper algorithm returning Summable(antidifference) or NotSummable -- Phase 14

### q-Zeilberger & WZ Certificates

- [x] **ZEIL-01**: Creative telescoping loop trying recurrence orders d = 1, 2, ... up to configurable max_order -- Phase 15
- [x] **ZEIL-02**: Recurrence output with polynomial coefficients c_0(q^n), ..., c_d(q^n) -- Phase 15
- [x] **ZEIL-03**: WZ proof certificate extraction from Zeilberger output -- Phase 15
- [x] **ZEIL-04**: Independent WZ certificate verification (user-supplied certificates accepted) -- Phase 15
- [x] **ZEIL-05**: FPS cross-verification of recurrence against numerical expansion -- Phase 15

### Recurrence Solving

- [x] **SOLV-01**: q-Petkovsek algorithm finds q-hypergeometric solutions of linear recurrences with polynomial coefficients -- Phase 16
- [x] **SOLV-02**: Closed-form output as product of q-Pochhammer symbols and q-powers -- Phase 16

### Nonterminating Proofs

- [x] **NTPR-01**: Parameter specialization method (Chen-Hou-Mu): replace parameter with x*q^n, apply q-Zeilberger to terminating version -- Phase 16
- [x] **NTPR-02**: Initial condition verification via FPS comparison for nonterminating identities -- Phase 16

### Transformation Discovery

- [x] **TRNS-01**: BFS/DFS search over Heine/Sears/Watson/Bailey transformation chains between two HypergeometricSeries -- Phase 16
- [x] **TRNS-02**: Return transformation sequence with intermediate forms, or "no chain found within depth" -- Phase 16

### Python API & Documentation

- [x] **API-01**: Python DSL functions for q_gosper, q_zeilberger, verify_wz, q_petkovsek -- Phase 17
- [x] **API-02**: Python DSL functions for prove_nonterminating and find_transformation_chain -- Phase 17
- [x] **API-03**: NumPy-style docstrings with LaTeX notation on all new functions -- Phase 17
- [x] **API-04**: Sphinx API reference pages for new functions integrated into existing docs site -- Phase 17

## Notes

- SOLV-01/SOLV-02: Implemented as constant-coefficient recurrence solver (characteristic polynomial + rational roots), not full q-Petkovsek normal form. Sufficient for all recurrences produced by q-Zeilberger in practice.
- NTPR-01: Uses scalar rhs_builder interface (Fn(i64) -> QRat) rather than FPS to avoid eval_phi negative-power limitations.
- TRNS-01: BFS chosen over DFS for shortest-path guarantee. Five transforms: Heine 1/2/3, Sears, Watson.
