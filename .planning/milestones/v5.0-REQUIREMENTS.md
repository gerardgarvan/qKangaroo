# Requirements: q-Kangaroo v5.0 (Archived)

**Defined:** 2026-02-21
**Archived:** 2026-02-22
**Status:** All 21 requirements satisfied
**Core Value:** Maximum Maple compatibility -- researchers can run their Maple code in q-Kangaroo with minimal modification

## v5.0 Requirements

### Bug Fixes

- [x] **BUG-01**: Division by exact polynomial (POLYNOMIAL_ORDER sentinel) completes in bounded time instead of hanging -- `1/aqprod(q,q,5)` and `q^n/aqprod(q,q,n)` inside for-loops work correctly

### Language Features

- [x] **LANG-01**: `while...do...od` loops execute with boolean/comparison conditions
- [x] **LANG-02**: List literals `[a, b, c]` as first-class values with display and indexing `L[i]`
- [x] **LANG-03**: Unicode operator resilience -- `^`->`^`, `-`->`-`, `*`->`*` accepted by parser
- [x] **LANG-04**: `print(expr)` displays intermediate results during loops/procedures

### Series & Polynomial Functions

- [x] **SERIES-01**: `coeff(f, q, n)` extracts the coefficient of `q^n` from a series
- [x] **SERIES-02**: `degree(f, q)` returns the degree of a polynomial/series
- [x] **SERIES-03**: `numer(f)` and `denom(f)` extract numerator/denominator of rational expressions

### Summation & Iteration Functions

- [x] **ITER-01**: `add(expr, i=a..b)` computes symbolic summation (Maple-style)
- [x] **ITER-02**: `mul(expr, i=a..b)` computes symbolic product (Maple-style)
- [x] **ITER-03**: `seq(expr, i=a..b)` generates a list/sequence

### List & Expression Functions

- [x] **LIST-01**: `nops(expr)` returns the number of operands/elements
- [x] **LIST-02**: `op(i, expr)` extracts the i-th operand/element
- [x] **LIST-03**: `map(f, list)` applies function f to each element
- [x] **LIST-04**: `sort(list)` sorts list elements

### Utility Functions

- [x] **UTIL-01**: `modp(a, p)` and `mods(a, p)` for modular arithmetic
- [x] **UTIL-02**: `type(expr, t)` checks expression type (integer, series, list, etc.)
- [x] **UTIL-03**: `evalb(expr)` evaluates boolean expression
- [x] **UTIL-04**: `cat(s1, s2, ...)` concatenates strings/names

### Documentation

- [x] **DOC-01**: Help entries and tab completion for all new functions/keywords
- [x] **DOC-02**: PDF manual chapter documenting v5.0 additions

## Out of Scope

| Feature | Reason |
|---------|--------|
| zqfactor (bivariate factoring) | Requires fundamental bivariate polynomial infrastructure |
| Full Maple parser compatibility | Goal is q-series research code, not arbitrary Maple programs |
| GUI / IDE integration | CLI and library only |
| convert() family | Too broad; specific conversions added as needed |
| assume() / assumptions | Complex symbolic engine feature beyond scope |
